# GMI — Generic MACOS Interface

GMI is a MATLAB MEX wrapper (`GMI.mexa64`) around the SMACOS Fortran
library that lets MATLAB scripts run a MACOS optical prescription,
apply per-element perturbations (rigid-body, Zernike, MonZern,
gridded, deformable mirror, …), and pull back OPD / PIX / SPOT /
complex-EF arrays for analysis. Built from the macos cmake tree's
`libsmacos.a`; intended as the "control system" entry point for
sensitivity computations, calibration loops, and closed-loop
wavefront sensing demos.

## Layout

| File / dir | Purpose |
|---|---|
| `GMI.F` | `MODULE GMI_mod` containing `GMI_DVR` (work routine) and helpers (`ObtainNominalSettings`, `SetToNominalSettings`, `ApplyPerturbationToOpticalSystem`, …) |
| `GMIG.F` | MEX gateway (`SUBROUTINE mexfunction`); validates `PRHS`, copies into Fortran arrays, calls `GMI_DVR` |
| `GMI.inc` | Compile-time array sizes (`numseg`, `mzern`, `mpzern`, `mprb`, `mpgrid`, …) |
| `GMI.mexa64` | Built MEX binary (gitignored; produced by `Makefile`) |
| `Makefile` | Standalone build (auto-detects MATLAB under `/usr/local/MATLAB`) |
| `call_GMI.m` | MATLAB wrapper — packs a `param` struct into `pflg` and invokes the MEX |
| `optiixInit_jzlou.m` | Reference `param` initializer (Optiix Rx) |
| `test_gmi.m` | Sample driver (rb sensitivity loops) |
| `optiixonaxisz1_v4_pmsm_met.in` | Optiix prescription used by `test_gmi.m` |
| `ff_pie.in` | FreeForm pie-segmented test prescription |
| `test_ff/` | Self-contained FreeForm sensitivity test directory (see below) |
| `CLAUDE.md` | Architecture / mex-arg / pflg-layout / nominal-restore reference |

## Build

Standalone `Makefile`, outside the macos cmake tree. Links pre-built
libraries from `~/dev/macos/build_release_giza/`:
- `libsmacos.a`, `libnpsol.a`, plus MATLAB `libmex` / `libmx`
- Consumes `.mod` files from `build_release_giza/mod_smacos/`

Entry points (in `~/dev/macos/`):
- `source ./makegmi.sh` — GMI MEX only (requires macos+smacos pre-built)
- `source ./makejoint.sh` — macos + smacos + smacos_dvr + GMI

`make` doesn't see `.inc` includes, so when only `GMI.inc` changes the
`.o` files don't rebuild. Force a rebuild:
```
rm -f GMI.o GMIG.o
source ~/dev/macos/makegmi.sh
```

## Quick start (MATLAB)

```matlab
optiixInit_jzlou;          % populate `param` and constants
prb = zeros(mprb,1);
[PIX,CE,OPD,OPDMask,SPOT,WFE,c,metMeas,USER] = ...
   call_GMI(prb, 0, 0, 0, 0, 0, ...
            param.pimg, InfFcnZern, InfFcnGrid, param);
imagesc(OPD); colorbar
```

`call_GMI` arguments: `(prb, pzern, pgrid, pdm, pfa, prad, pimg,
InfFcnZern, InfFcnGrid, param[, winfil])`. Pass scalar `0` for any
perturbation channel you don't use.

## Perturbation channels

`ApplyPerturbationToOpticalSystem` (in `GMI.F`) iterates three
element-list × coefficient-vector pairs:

| Channel | Surface list | Coefficient vector | What it writes |
|---|---|---|---|
| Zernike | `param.zernSrf` | `pzern` | `ZernCoef(iNode, iElt)` for `iNode = 4..nzern+3`; **forces** `SrfType(iElt)=8` |
| MonZern (FreeForm) | `param.monzernSrf` | `param.pmonzern` | `MonZernCoef(iNode, iElt)` for `iNode = 4..nmonzern+3`; **does not change SrfType** (caller must already have FreeForm/14) |
| Rigid body | `param.rbSrf` | `prb` | calls `PERTURB`/`GPERTURB` via SMACOS; modifies `TElt`, `psiElt`, `vptElt`, `pMon/xMon/yMon/zMon`, and (on FreeForm) `pFF/xFF/yFF/zFF`, `pData/xData/yData/zData` |

All three loops use `DO j=1, jXSrf-1` × `DO i=1, iXSrf` — the **last
column** of the surface list is reserved for a flag (e.g. for `rbSrf`
it's the global(0) / local(1) frame). A single-column list silently
disables the channel; for `rbSrf` write `[(1:N)' zeros(N,1)]`, not
`[1:N]'`.

## test_ff/ — FreeForm sensitivity end-to-end test

Self-contained directory exercising the FreeForm Mon/FF perturbation
channels against pie- and hex-segmented prescriptions generated by
SegMirMaker.

```
test_ff/
├── test_gmi_ff.m         # main driver: nominal OPD → mask → dwdx → dwdz
├── gmi_Init_ff.m         # param initializer for FreeForm tests
├── call_GMI.m            # local copy (in sync with the canonical one)
├── ff_pie.in             # SegMirMaker-generated 7-segment pie parent
├── ff_hex1.in, ff_hex2.in# hex-segmented variants
├── ff_pie_preswap.in     # pre-fix copy (regression reference for
│                         #   the SegMirMaker psi-flip change)
├── ray_check.m           # per-segment ray-trace sanity check;
│                         #   compare ray end point vs RptElt
├── flat.txt              # all-zero grid data referenced by parents
├── zern41em5z155em3.txt  # Zernike data file
├── *.in                  # additional parent prescriptions
├── macos_param.txt       # MACOS sizing
└── macos_glass_list.txt  # MACOS glass tables
```

Run from MATLAB:
```matlab
cd ~/dev/MACOS_resources/GMI/test_ff
test_gmi_ff
```

Outputs (`.mat` cache files, all gitignored):
- `OPDMask_g.mat` — global wavefront mask cached after first run
- `ff_pie_dwdx_urad_um.mat` — rigid-body wavefront sensitivities
- `ff_pie_dwdz_urad_um.mat` — MonZern surface-deformation sensitivities

After fixing rb plumbing or any state-leak in `GMI.F`, **delete
`OPDMask_g.mat`** before rerunning — otherwise the cached (bad) mask
is reused.
